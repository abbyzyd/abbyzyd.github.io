Docker基本命令

参见资料：docker基本命令

获取镜像

docker pull <镜像名称>

例：docker pull hello-world

镜像库查找镜像

docker search <镜像关键字>

例：docker search mysql

查看镜像

docker images

删除镜像

docker rmi <镜像Id>

启动容器

启动后进入命令行模式

docker run -it <镜像名称>[:<标签>] /bin/bash

例：docker run it centos:7.8.25 /bin/bash

后台启动

docker run -itd --name <容器名称> <镜像名称>[:标签] /bin/bash

端口映射

docker run -itd --name <容器名称> -p <宿主机端口>:<容器端口> <镜像名称>[:标签] /usr/sbin/init

开启root权限

docker run -itd --name <容器名称> --privileged=true <镜像名称>[:标签] /usr/sbin/init

实例

docker run -itd --name centos-mysql --privileged=true -p 2222:22 -p 3308:3306 centos:mysql-sshd2 /usr/sbin/init

查看容器

docker ps -a

启动/关闭/重启容器

docker start <容器Id>

docker stop <容器Id>

docker restart <容器Id>

进入容器

推荐：docker exec -it <容器名称/容器Id> /bin/bash，退出容器，容器不会关闭

docker attach <容器名称/容器Id>，退出容器，容器自动关闭

退出容器

exit

删除容器

docker rm -f <容器Id>

打包容器导成镜像

docker commit <容器ID> <镜像名称/标签>

Dockerfile制作镜像

vi Dockerfile

docker build -t <镜像名称>[:标签] .

docker save -o <目标tar文件> <镜像名称>:<版本>

docker save -o /home/zhyd/centos7.tar centos:7

导入镜像

docker import <tar文件> <目标镜像名称>:<目标版本>

部署Hexo+GitPage部署

部署Hexo+GitPage

Hexo安装

安装git

检查版本

git --version

安装 node

检查版本

node -v

安装 hexo

在本地建立一个文件夹（博客的目录），例如：E:\git\blog，进入这个文件夹，鼠标右键选择Git Bash Here

npm i -g hexo

hexo init

npm install

查看 hexo 版本

hexo version

Git建立博客仓库

仓库名必须为：用户名+github.io

Git设置SSH

查看配置的邮箱

git config --list

生成SSH公私钥

ssh-keygen -t rsa -C "邮箱地址"

会在C:\Users\${用户名}\.ssh下生成id_rsa和id_rsa.pub两个文件

配置公钥

登录github的个人账户，设置ssh公钥

测试是否授权生效

ssh git@github.com

出现You’ve successfully authenticated即成功

hexo 常用命令

下载主题

git clone https://github.com/theme-next/hexo-theme-next themes/next

修改_config.yml对应的主题 theme: next

新建Post

在blog目录下右键Git Bash here

hexo n “my fisrt blog”

在source\_posts文件夹下会出现my fisrt blog.md，在里面编写内容即可

删除Post

先删掉db.json这个文件，把对应的md文件也删掉，重新执行

hexo clean

hexo g

hexo d

清除数据

hexo clean

生成静态页面

hexo g

部署博客

在_config.yml的deploy下面配置

deploy:
type: git
repo: git@github.com:用户名/用户名.github.io.git
branch: master

然后执行hexo d
